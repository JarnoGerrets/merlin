# Vault Maintenance Checklist

Use this checklist after meaningful vault or implementation work.

## Link Health

- [ ] No broken wiki links.
- [ ] No ambiguous duplicate note names for curated notes.
- [ ] Renamed notes have updated backlinks.

## Status Health

- [ ] No feature marked implemented without Verified Against Code.
- [ ] Future/blocked items clearly explain why.
- [ ] Current-state notes reflect recent work.

## Writeback Health

- [ ] Code changes have an Agent Run report.
- [ ] Changelog updated.
- [ ] Progress report updated.
- [ ] Feature notes updated.
- [ ] Code Atlas updated for changed files/classes.

## Prompt Health

- [ ] Implementation prompts reference prompt bundles.
- [ ] Implementation plans list required bundles/extensions.
- [ ] Obsolete prompts are marked obsolete.
- [ ] Plans have lifecycle status.

## Bug Health

- [ ] New bugs are in `09_Bugs`.
- [ ] Fixed bugs include validation.
- [ ] Bug notes link to affected features/code atlas.
