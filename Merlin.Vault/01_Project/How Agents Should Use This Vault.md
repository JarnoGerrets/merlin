# How Agents Should Use This Vault

1. Read `Merlin.Vault/00_Index.md` first.
2. Read the relevant feature note.
3. Read the relevant architecture note.
4. Check dependencies and non-goals.
5. Check whether the requested work is Ready, Blocked, Future, or already Implemented.
6. Never implement Future/V2 items unless explicitly asked.
7. Never bypass safety/confirmation systems.
8. Update the vault after implementation.
9. Add a short implementation report and link it from the feature note.
10. Keep current implementation and planned architecture separate.

## Agent Rule

Current implementation notes describe what code does today. Roadmaps and planned sections describe possible future work. Do not treat a plan as implemented until code and tests confirm it.
