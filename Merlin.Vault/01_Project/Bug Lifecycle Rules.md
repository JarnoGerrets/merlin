# Bug Lifecycle Rules

## Statuses

- open
- investigating
- fixed
- verified
- wont-fix

## Rules

1. A bug mentioned during an agent run must be added to `09_Bugs` if it is still relevant.
2. A fixed bug is not verified until validation is documented.
3. Bug reports should link to affected feature notes, code atlas notes, and agent runs.
4. Do not leave bugs only in chat or final responses.
5. If a bug is actually a design limitation, mark it clearly as fragility/limitation.
